## Задача

Функция становится полезной, когда принимает данные через параметры и возвращает результат через `return`. А ещё параметру можно задать значение по умолчанию — на случай, если аргумент не передали. Сегодня набиваешь руку на этом.

```
function add(a, b = 10) {  return a + b  }
         параметры  ↑default      ↑возврат

add(2, 3) → 5     add(2) → 12 (b взял дефолт)
вызов без аргумента → параметр = undefined (или default)
```

## Что нужно сделать

В `params.js` написать несколько маленьких функций с параметрами и `return`: сумма двух чисел; площадь прямоугольника; приветствие с временем суток, где время по умолчанию `'день'`. Вызвать каждую и с полным набором аргументов, и с неполным (чтобы сработал default или проявился `undefined`).

## Критерии приёмки

* Минимум три функции с параметрами и `return`, каждая вызвана несколько раз с разными аргументами.
* Есть параметр со значением по умолчанию; показано, что происходит при вызове без него.
* Понятна разница между параметром (в объявлении) и аргументом (при вызове), и что вернёт функция без `return`.
